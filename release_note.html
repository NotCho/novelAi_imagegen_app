NAIApp 1.4.0 업데이트 소식

이번 업데이트에서는 베타 피드백을 바탕으로 와일드카드 관리 안정성, 저장 경로 설정, 프리셋/입력 UX, 모델별 레퍼런스 호환성을 함께 개선했습니다.

[와일드카드] 큰 파일도 더 안정적으로

  • 와일드카드 본문을 앱 내부 .txt 파일로 저장하도록 변경했습니다.
  • 기존에 저장된 와일드카드는 앱 실행 시 새 저장 방식으로 자동 마이그레이션됩니다.
  • 큰 와일드카드 파일 때문에 앱이 다시 켜지지 않던 상황을 줄였습니다.
  • 편집과 미리보기에서 option: weight 가중치 형식이 유지됩니다.

[저장] 이미지 저장 경로 설정

  • 설정 화면에서 이미지 저장 경로를 선택할 수 있습니다.
  • 저장 경로를 초기화하면 기존처럼 기본 갤러리 저장으로 돌아갑니다.
  • Android에서는 MediaStore를 사용해 선택한 폴더명 기준으로 저장되도록 개선했습니다.
  • PNG/WebP 저장과 이미지 메타데이터 보존 흐름은 기존 방식과 동일하게 유지됩니다.

[프리셋/입력] 사용 흐름 개선

  • 프리셋 새 저장, 덮어쓰기, 이름 변경, 삭제 흐름을 정리했습니다.
  • 프리셋 더보기 버튼의 터치 영역을 확인하기 쉽도록 테두리를 추가했습니다.
  • 프롬프트 입력창과 프롬프트 변환의 새 태그 추가 창에서 키보드가 열릴 때 화면이 과하게 재배치되는 문제를 줄였습니다.
  • 입력창 높이 계산을 조정해 프리셋 입력창과 더 일관되게 보이도록 다듬었습니다.

[모델 호환성] 레퍼런스 기능 요청 정리

  • Vibe Transfer: v4 이상 모델에서만 사용할 수 있도록 정리했습니다.
  • Character Reference: v4.5 이상 모델에서만 사용할 수 있도록 정리했습니다.
  • UI에서 비활성화된 레퍼런스 기능이 실제 요청 payload에 남아 400 오류를 만드는 문제를 수정했습니다.

[개발자] 개발자 메모

이번 버전부터 1.3.99 베타에서 확인된 주요 피드백을 정식 업데이트에 반영했습니다. 특히 와일드카드 저장 방식이 바뀌었기 때문에, 첫 실행 시 기존 와일드카드 데이터가 내부 txt 파일로 정리될 수 있습니다.

사용 중 저장 경로나 와일드카드 파일 관련 문제가 계속 보인다면 사용 중인 Android 버전, 선택한 저장 위치, 와일드카드 파일 크기를 함께 알려주세요.

[다운로드] 다운로드

앱 내 업데이트 알림을 통해 1.4.0 버전을 설치할 수 있습니다.
